关闭 sony 网络错误

这条 adb shell settings put global captive_portal_mode 0 命令用于修改 Android 设备的全局设置，具体来说，它控制 “强制使用网络登录检测”（Captive Portal Detection） 的行为。

解析：

  • adb shell：进入 Android 设备的 shell 命令行环境。
  • settings put global：修改全局系统设置（Global Settings）。
  • captive_portal_mode：表示 Android 设备的强制网络登录检测模式。
  • 0：表示 禁用 网络门户（Captive Portal）检测。

captive_portal_mode 的值含义：

值 含义
0 禁用 网络门户检测（不会检查网络是否需要 Web 认证）
1 正常模式（默认，Android 会检测网络是否需要 Web 认证）
2 假定网络总是可用（即使需要认证也认为是正常的）

作用：

  • 如果你连接到一个需要 Web 认证（如 Wi-Fi 认证页面）的网络，通常 Android 会弹出一个登录页面（Captive Portal）。
  • 但如果你设置 captive_portal_mode 0，Android 设备将不会再检测网络门户，也不会弹出 Wi-Fi 登录页面，即使连接的是需要认证的网络。

适用场景：

  • 适用于不希望 Android 进行网络检测的情况，比如某些企业 Wi-Fi 或 IoT 设备。
  • 在部分情况下，可以用于加快 Wi-Fi 连接过程，避免 Android 误判网络状态。
